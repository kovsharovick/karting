# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений.
> Заполняется после ответов заказчика ([customer-questions.md](customer-questions.md)).

## Описание домена

**Бизнес.** «Апекс» — уличный картинг-центр на окраине города, работающий третий сезон.
Центр организует **заезды** (групповое катание) по трассе. Заезд длится 15–20 минут и включает
инструктаж перед стартом. Доступны две **конфигурации трассы**: короткая — для новичков, и
длинная — для опытных. Каждый заезд ведёт **маршал-инструктор** (сейчас их пятеро, летом
нанимаются подработчики).

**Процесс.** Владелец (Денис) заранее формирует **расписание на неделю вперёд**: для каждого
заезда задаются дата и время старта, конфигурация трассы, назначенный маршал и число доступных
карт. Парк центра — **14 картов**; часть из них имеет ограничение скорости и используется на
новичковых заездах, где дополнительно действует потолок **не более 8 человек** в группе. Клиент
видит свободные **слоты** заездов и **записывается**, указывая, поедет он со своей экипировкой
(шлем, подшлемник) или возьмёт её **напрокат**; при этом возможна **групповая бронь** — одной
записью на себя и на несколько участников сразу. До заезда клиент может **отменить** запись —
до 1 часа до старта отмена свободна, позже — тоже возможна, но с предупреждением и учётом
поздней отмены в статистике центра. После состоявшегося заезда клиент может **оценить маршала**
(рейтинг которого виден другим клиентам заранее, в карточке слота). Сейчас запись ведётся
вручную в Telegram и на доске маркером, что приводит к двойным записям и путанице по выходным.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на заезды (просмотр
и фильтрация слотов, групповое бронирование, выбор экипировки, отмена, свои брони, оценка
маршала после заезда), исключив ручную путаницу и двойные брони. Формирование расписания и
работу маршалов/владельца обеспечивает **существующая инфраструктура**; приложение потребляет
её данные через API.

**Ключевые объекты домена:**

- **Заезд / слот** — конкретный выезд группы: дата/время старта, конфигурация трассы, маршал (с
  его средним рейтингом), всего и свободно мест, требования по возрасту/росту, итоговая
  стоимость. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Конфигурация трассы** — короткая (новичковая, потолок 8 человек) или длинная (опытная);
  справочные данные слота, read-only для клиента; отдельного признака «клиент-новичок» не
  существует — уровень определяется выбранной трассой.
- **Маршал-инструктор** — ведёт заезд; для клиента — справочные данные слота и объект оценки
  постфактум (read-only).
- **Карт** — инвентарь (всего 14, часть — с ограничением скорости для новичковых заездов); может
  выдаваться в связке с прокатной экипировкой.
- **Экипировка** — шлем и подшлемник; выбор «своя» / «прокат» на этапе записи, прокат ограничен
  свободным фондом, приходящим из API.
- **Бронь (запись)** — резерв одного или нескольких мест клиентом на конкретный слот: число мест,
  выбор экипировки, статус (активна / отменена клиентом / отменена центром / завершена).
- **Оценка маршала** — необязательная, разовая (без редактирования) обратная связь клиента после
  завершённого заезда; средний рейтинг маршала виден клиентам в карточке слота.
- **Клиент** — записывается на заезды (в т.ч. с гостями), отменяет запись, оценивает маршала.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское мобильное приложение и API для него.**
> Роли Маршал и Владелец/Администратор обслуживаются **существующей инфраструктурой**
> (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость центра, катается на картинге. Пользуется с телефона. | Увидеть и отфильтровать свободные слоты, записаться (себя и гостей), выбрать свою/прокатную экипировку, видеть свои брони, отменить запись, оценить маршала после заезда, получать напоминания. | **Да** |

> Маршал и Владелец/Администратор взаимодействуют с системой через **уже реализованные**
> интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
> ими данные (слоты, конфигурации трассы, маршалы) через API.

## Ограничения

**Бизнес-ограничения**

- Всего **14 картов** — общий потолок инвентаря; часть с ограничением скорости выделена под
  новичковые заезды.
- Новичковая группа (короткая трасса) — **не более 8 человек**; лимит определяется
  конфигурацией трассы слота, а не профилем клиента.
- Лимит мест слота = `min(свободные карты, потолок конфигурации трассы)`.
- **Политика отмен:** бесплатная отмена не позднее чем за **1 час** до старта; поздняя отмена
  (<1 ч) разрешена, но сопровождается предупреждением в UI и учитывается в статистике поздних
  отмен на стороне владельца — без блокировок или штрафов на стороне клиента.
- **Оплата офлайн** (наличные / перевод на карту) — приложение показывает итоговую стоимость
  заезда и проката из данных API, но реального платёжного шлюза нет.
- **Групповая бронь** — одна запись может включать несколько мест (клиент + гости), без сбора
  персональных данных гостей.
- **Возрастные/ростовые требования** — отображаются как предупреждение на карточке слота;
  приложение их не валидирует, финальная проверка допуска — за маршалом на месте.
- **Период списка слотов по умолчанию — 7 дней (R-027).** По умолчанию клиенту показываются
  слоты на **ближайшие 7 дней** от текущего момента — совпадает с горизонтом планирования
  расписания. Более длинный период — через **фильтр дат**. При отсутствии расписания на
  ближайшие дни показывается **empty state** («Пока нет доступных заездов»).
- **Отмена заезда центром / по погоде (R-008).** Если заезд отменяет центр (организатор или
  из-за погоды), бронь на стороне клиента **не удаляется**, а получает статус **«Отменён
  центром»** с указанием причины; клиенту приходит **push-уведомление**. **Повторная запись на
  отменённый слот запрещена.** Само действие отмены слота выполняется в существующей
  инфраструктуре (вне скоупа клиентского приложения).

**Технические / платформенные**

- **Mobile-first**: клиентское приложение рассчитано на телефон (используется прямо у трассы).
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты,
  конфигурации трасс и маршалы доступны только для чтения.
- **Авторизация** — телефон + одноразовый код в SMS, без паролей.
- **Граница серверной интеграции (R-004).** Детали серверной реализации — транзакционность,
  SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**.
  Существующий бэкенд — **black-box источник истины**: гарантия «**0 двойных броней**»
  обеспечивается на его стороне (атомарная проверка свободных картов при бронировании). Со
  стороны клиента граница закрывается **контрактом API**: приложение полагается на ответы
  бэкенда и корректно обрабатывает отказ, если мест уже нет.
- **Данные существующей инфраструктуры (R-015).** Это **учебный/тестовый проект**, исторических
  данных и легаси-схемы нет. **Канонической схемой считается контракт API**: бэкенд отдаёт все
  поля контракта, включая прокатный тариф на экипировку, конфигурацию трассы, адрес центра и
  место сбора, свободный прокатный фонд экипировки. **Миграция/дозаполнение старых данных вне
  скоупа** — отдельных требований к ним не формулируем.

**Проект**

- **Срок** — к старту высокого сезона, ~2 месяца (жёсткая цель).
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» — позже.

**Границы скоупа**

Скоуп — **клиентское мобильное приложение и API для него**. Формирование расписания, роли
маршала и владельца, инвентарь картов, логика двойного бронирования, отмена заезда — это
**существующая инфраструктура**, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента по телефону + SMS-код | Роли Маршала и Владельца/Администратора, админ-экран |
| Просмотр и **фильтрация** списка слотов (дата, конфигурация трассы, наличие мест, маршал) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись, в т.ч. групповая (несколько мест за раз) | Назначение маршалов, управление парком картов |
| Выбор «своя / прокатная экипировка», контроль прокатного фонда | Действие отмены слота центром/по погоде — само действие выполняется в инфраструктуре |
| Список своих бронирований | Отметка явки/неявки клиента |
| Отмена записи с правилом 1 часа | Онлайн-оплата; денежные штрафы за поздние отмены |
| Итоговая цена заезда и проката (оплата офлайн) | Программа лояльности для постоянных клиентов |
| Оценка маршала после завершённого заезда (необязательная, разовая) | Автоматический учёт погоды по прогнозу |
| Отображение среднего рейтинга маршала в карточке слота | Ручная отметка «оплачено» |
| Push-напоминание перед стартом + push при отмене центром (R-008) | Дополнительные каналы уведомлений (SMS/email), подтверждения брони — по решению позже |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Заезд / слот** | Запланированный групповой выезд: дата/время старта, конфигурация трассы, маршал, всего и свободно мест. |
| **Конфигурация трассы** | Вариант трассы: короткая (новичковая, потолок 8 человек) или длинная (опытная, до 14 по числу картов). |
| **Группа** | Состав клиентов, записанных на один слот. |
| **Карт** | Единица инвентаря центра; всего 14; часть — с ограничением скорости для новичковых заездов. |
| **Экипировка** | Шлем и подшлемник; своя или клубная (прокатная). |
| **Прокат** | Выдача клубной экипировки клиенту, у которого нет своей; ограничен свободным фондом на слот. |
| **Своя экипировка** | Клиент приезжает со своим шлемом/подшлемником; занимает место в группе, но не прокатный фонд. |
| **Запись (бронь)** | Резерв одного или нескольких мест на конкретный слот: число мест, выбор экипировки, статус. |
| **Отмена** | Снятие записи. Клиентом: до 1 часа до старта — свободно; после — возможна, но с предупреждением и учётом в статистике поздних отмен. Центром (организатором / по погоде): бронь переходит в статус «Отменён центром» с причиной, клиент получает push, повторная запись на этот слот запрещена. |
| **Отмена центром** | Отмена заезда организатором или из-за погоды (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменён центром» + причина; отправляется push-уведомление; повторная запись на отменённый слот недоступна. |
| **Оценка маршала** | Необязательная разовая (без редактирования) обратная связь клиента после завершённого заезда; средний рейтинг маршала виден клиентам в карточке слота. |
| **Администратор / владелец** | Денис — управляет расписанием и маршалами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту высокого сезона. |
