# Бизнес-требования

> Этап 2. Зачем бизнесу продукт: цели, метрики успеха, ограничения верхнего уровня.

> **Скоуп — клиентское мобильное приложение и API для него.** Формирование расписания, роли
> маршала и владельца обслуживаются **существующей инфраструктурой** и в этот список не входят.

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| BR-1 | Устранить ручную путаницу и двойные записи, возникающие при ведении записи через Telegram и на доске маркером; обеспечить единый учёт броней без конфликтов по местам. | Must | [domain-description.md](../1-elicitation/domain-description.md) → «Процесс», «Цель клиентского приложения» |
| BR-2 | Дать клиентам возможность самостоятельно записываться на заезды онлайн: просмотр и фильтрация слотов, групповое бронирование, выбор экипировки, отмена, просмотр своих броней, оценка маршала после заезда. | Must | [domain-description.md](../1-elicitation/domain-description.md) → «Цель клиентского приложения» |
| BR-3 | Запустить клиентское приложение к старту высокого сезона (~2 месяца) при ограниченном бюджете, начав с минимально необходимого набора функций (MVP). | Must | [domain-description.md](../1-elicitation/domain-description.md) → «Проект», «Границы скоупа» |
| BR-4 | На старте оплата остаётся офлайн (наличные / перевод на карту); приложение показывает итоговую стоимость заезда и проката из API, но не содержит платёжного шлюза. | Must | [domain-description.md](../1-elicitation/domain-description.md) → «Ограничения» → «Оплата офлайн» |
| BR-5 | Клиентское приложение — mobile-first (использование с телефона у трассы); данные о слотах, конфигурациях трасс и маршалах получает из существующего бэкенда через API (read-only). | Must | [domain-description.md](../1-elicitation/domain-description.md) → «Технические / платформенные», «Границы скоупа» |

## Ограничения верхнего уровня

- **Инвентарь.** Всего 14 картов; часть с ограничением скорости — для новичковых заездов. ([domain-description.md](../1-elicitation/domain-description.md) → «Бизнес-ограничения»)
- **Потолок группы.** Новичковая конфигурация (короткая трасса) — не более 8 человек; лимит мест слота = `min(свободные карты, потолок конфигурации трассы)`. ([domain-description.md](../1-elicitation/domain-description.md) → «Бизнес-ограничения»)
- **Политика отмен клиентом.** Бесплатная отмена не позднее чем за 1 час до старта; поздняя отмена (<1 ч) разрешена с предупреждением в UI и учётом в статистике центра — без блокировок и штрафов для клиента. ([domain-description.md](../1-elicitation/domain-description.md) → «Бизнес-ограничения»)
- **Групповая бронь.** Одна запись может включать несколько мест (клиент + гости) без сбора персональных данных гостей. ([domain-description.md](../1-elicitation/domain-description.md) → «Бизнес-ограничения»)
- **Вне скоупа клиентского приложения.** Расписание, роли маршала и владельца, управление парком картов, онлайн-оплата, штрафы за поздние отмены, программа лояльности, автоматический учёт погоды, отметка явки/оплаты. ([domain-description.md](../1-elicitation/domain-description.md) → «Границы скоупа»)
