# Функциональные требования

> Этап 2. Что система должна делать (функции, поведение).
> Не путать с нефункциональными ([non-functional-requirements.md](non-functional-requirements.md)).

Приоритеты по MoSCoW: **Must** — обязательно к запуску; **Should** — желательно при наличии времени; **Won't** — вне границ скоупа.

> **Скоуп — клиентское мобильное приложение и API для него.** Слоты, конфигурации трасс и
> маршалы создаются в существующей инфраструктуре и доступны клиенту только для чтения через API.
