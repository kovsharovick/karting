## 1. Продукт и аудитория

**«Апекс»** — клиентское мобильное приложение для самостоятельной записи на групповые заезды
картинг-центра. Заменяет запись через Telegram и на доске маркером.

**Единственная роль — «Клиент».** Маршал и владелец в приложение не входят. Слоты, конфигурации
трасс и маршалы — read-only из API. Оплата — **офлайн**; приложение показывает цену и фиксирует
запись, онлайн-оплаты нет.

**Контекст использования:** клиент у трассы, нередко на солнце, спешит перед заездом. Нужны
крупные элементы, высокий контраст и минимум шагов (NFR-1).

## 2. Дизайн-принципы

| # | Принцип | Источник | Что это значит для макета |
|---|---------|----------|---------------------------|
| P1 | **Mobile-first у трассы** | NFR-1 | Крупные тач-зоны, высокий контраст, читаемость на ярком солнце, минимум мелкого текста. |
| P2 | **Короткий путь к записи** | BR-2, домен | От списка до подтверждения — минимум экранов; не добавлять необязательных шагов. |
| P3 | **Минимальный порог входа** | FR-1, NFR-5 | Вход по телефону + SMS-код, без пароля. |
| P4 | **Честность и спокойствие** | UC-3, UC-4 | Ошибки и правила (места, прокат, 1 час до отмены) объясняются понятно; штрафов нет. |
| P5 | **Только свои данные** | NFR-6 | Клиент видит лишь свои брони; справочники — read-only. |
| P6 | **Доверие к API** | NFR-3, R-004 | При отказе бронирования UI показывает актуальную доступность; двойных броней быть не должно. |

## 3. Структурные токены (без бренда)

### 3.1 Тач-зоны и размеры
- Минимальный размер интерактивного элемента — **≥ 44–48 pt** по меньшей стороне.
- Основной CTA — во всю ширину контентной области.
- Между кликабельными элементами — отступ, исключающий промахи.

### 3.2 Контраст и читаемость
- Контраст текста к фону — не ниже **WCAG AA** (ориентир).
- Состояния не передаются **только цветом** — дублируются иконкой/текстом/формой.
- Важные числа (свободно мест, цена, время старта, рейтинг маршала) — крупные и контрастные.

### 3.3 Типографическая иерархия
- **Заголовок экрана** → **Заголовок секции/карточки** → **Основной текст** → **Caption**.
- Минимальный размер основного текста комфортен для чтения на улице.

### 3.4 Плотность и сетка
- Единая шкала отступов; контент — в одну колонку (mobile-first).

### 3.5 Иконки и индикаторы
- Иконки таб-бара и статусов сопровождаются текстом.
- Индикатор активных фильтров и бейджи статусов брони считываются с первого взгляда.

## 4. Каркас экрана и навигация

### 4.1 Базовый каркас
```
┌─────────────────────────────┐
│ Хедер (заголовок / назад)    │  ← фиксированный
├─────────────────────────────┤
│ Скролл-контент               │
├─────────────────────────────┤
│ Фикс. нижний CTA (если есть) │
└─────────────────────────────┘
│ Таб-бар (корневые экраны АЗ) │
└─────────────────────────────┘
```

### 4.2 Таб-бар (авторизованная зона)
Два верхнеуровневых раздела:
- **Заезды** ([SCR-002](../screens/SCR-002-slot-list.md)) — стартовая вкладка.
- **Мои брони** ([SCR-005](../screens/SCR-005-my-bookings.md)).

Таб-бар скрывается на вложенных экранах и bottom sheet. Каждая иконка с подписью.

### 4.3 Bottom Sheet (шторки BS-001 … BS-004)
- Высота — по контенту, не выше ~90% экрана; длинный контент скроллится внутри.
- Бэкдроп + закрытие по тапу вне шторки (кроме критичных подтверждений BS-003).
- Swipe-to-close + видимый «грабер» сверху.
- Действия-кнопки — в нижней части шторки.

### 4.4 Блок «Место сбора и адрес центра»

Сквозной информационный блок для [SCR-003](../screens/SCR-003-slot-card.md) и
[SCR-006](../screens/SCR-006-booking-details.md):

```
┌─────────────────────────────────┐
│  📍 Место сбора                  │
│  Площадка у павильона №2         │  ← meeting_point из API
│  Адрес центра: ул. …, г. …       │  ← address из API
└─────────────────────────────────┘
```

- Только **текстовое** отображение (карты в скоупе домена нет).
- Блок обязателен, если поля приходят из API (R-015).

## 5. Сквозной паттерн состояний экрана

| Состояние | Что показываем | Правило |
|-----------|----------------|---------|
| **Loading** | Скелетон/шиммер в форме будущего контента | Не пустой белый экран. |
| **Content** | Данные | Основной сценарий. |
| **Empty** | Заглушка + подсказка + действие (если уместно) | Текст «Пока нет доступных заездов» для пустого расписания (R-027). |
| **Error** | Заглушка + кнопка **«Обновить»** | Нейтральный тон. |

## 6. Tone of voice и общая микрокопия

**Тон:** простой, прямой, дружелюбный. Обращение на «вы». Штрафов и блокировок нет.

| Контекст | Текст |
|----------|-------|
| Оплата | «Оплата на месте: наличные или перевод на карту.» |
| Экипировка | «Своя экипировка» / «Прокат» (шлем и подшлемник) |
| Конфигурация трассы | «Короткая (новичковая)» / «Длинная (опытная)» |
| Правило отмены | «Отмена не позднее чем за 1 час до старта — место освобождается. Позже — отмена возможна, но место может остаться занятым. Штрафов нет.» |
| Поздняя отмена (итог) | «Поздняя отмена. Штраф не взимается.» |
| Отмена центром | «Заезд отменён центром» + `{причина}` |
| Возраст/рост | «Требования к участникам» + текст предупреждения; «Финальная проверка — у маршала на месте.» |
| Кнопка повтора | «Обновить» |
| Сетевая ошибка | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |

> **Раздельная модель доступности.** Места: `max_seats = min(свободные карты, потолок конфигурации трассы)`.
> Прокат: `rental_count ≤ free_rental_equipment`. Своя экипировка не расходует прокатный фонд.

> **Единый источник правила 1 часа.** Полные формулировки — только в этой таблице.
> [SCR-006](../screens/SCR-006-booking-details.md) и [BS-003](../screens/BS-003-cancel-confirm.md) **ссылаются** сюда.

## 7. Доступность

- Контраст — ориентир WCAG AA (§3.2).
- Тач-зоны ≥ 44 pt.
- Поддержка системного увеличения шрифта.
- Screen reader: labels у интерактивных элементов; статусы озвучиваются.
- Состояния не только цветом.

## 8. Сквозные функции

### 8.1 Push-уведомления (FR-20, FR-21)
- Напоминание перед заездом (Should).
- Уведомление при отмене центром (Must, R-008).
- Запрос разрешения на push — **после первой успешной записи** в [BS-002](../screens/BS-002-booking-success.md).
- [SCR-001](../screens/SCR-001-registration.md) разрешение **не** запрашивает.

### 8.2 Безопасность данных в UI (NFR-6)
- Только данные текущего клиента; телефон маскируется по усмотрению реализации.

## 9. Глоссарий UI

| Термин | Значение |
|--------|----------|
| **Заезд / слот** | Запланированный групповой выезд: дата/время, конфигурация трассы, маршал, места, цена. |
| **Конфигурация трассы** | Короткая (новичковая, ≤8 чел.) или длинная (опытная). |
| **Маршал** | Инструктор заезда; объект оценки после заезда. |
| **Экипировка** | Шлем и подшлемник: своя или прокатная. |
| **Бронь** | Резерв мест на слот со статусом: активна / отменена клиентом / отменена центром / завершена. |
| **Ранняя отмена** | ≥ 1 ч до старта → места освобождаются. |
| **Поздняя отмена** | < 1 ч до старта → предупреждение, учёт в статистике центра, штрафов нет. |
