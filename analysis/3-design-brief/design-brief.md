## Цель и контекст

**«Апекс»** — уличный картинг-центр. Клиентское приложение заменяет ручную запись через
Telegram и на доске маркером, устраняя двойные брони и путаницу с местами. Клиент самостоятельно
просматривает слоты заездов, фильтрует, записывается (себя и гостей), выбирает экипировку,
отменяет записи, оценивает маршала после заезда и получает напоминания. Платформа —
**mobile-first** (использование с телефона у трассы). Данные о слотах, конфигурациях трасс и
маршалах приходят из **существующего бэкенда через API** (read-only). Оплата — **офлайн**
(наличные / перевод); приложение показывает итоговую стоимость и фиксирует запись.

**Скоуп — только роль «Клиент».** Маршал и владелец работают через существующую инфраструктуру
и в приложение не входят.

**Что входит в MVP (из домена):**
- **Авторизация:** телефон + одноразовый SMS-код, без паролей (FR-1).
- **Слоты:** список на **7 дней** по умолчанию (R-027); фильтры по дате, конфигурации трассы,
  наличию мест, маршалу (FR-2, FR-3).
- **Запись:** групповая бронь без данных гостей; выбор «своя / прокатная» экипировка на место;
  лимит мест `min(свободные карты, потолок конфигурации трассы)`; прокат — отдельный фонд (FR-6–FR-11).
- **Отмена:** свободная до **1 часа** до старта; позже — с предупреждением, без штрафов (FR-15–FR-17).
- **Отмена центром (R-008):** статус «Отменён центром» + причина + push; повторная запись на слот запрещена (FR-18).
- **Оценка маршала:** необязательная, однократная, после завершённого заезда; средний рейтинг виден в карточке слота (FR-5, FR-19).
- **Push:** напоминание перед заездом (Should); уведомление при отмене центром (Must) (FR-20, FR-21).

Подробнее о продукте, аудитории и принципах — [00-foundations.md §1–2](00-foundations.md#1-продукт-и-аудитория).

## Роли и их задачи

| Роль | Что делает в приложении |
| :-- | :-- |
| **Клиент** | Просматривает и фильтрует слоты, записывается (себя + гости), выбирает экипировку, смотрит цену, отменяет записи, видит свои брони, оценивает маршала, получает push. |
| Маршал / Владелец | **Не входят в приложение** — работают через существующую инфраструктуру (вне скоупа). |

## Сквозные правила (foundations)

Единые для всех экранов требования вынесены в **[00-foundations.md](00-foundations.md)**:
дизайн-принципы, структурные токены (контраст, тач-зоны, иерархия — без бренда), каркас и
навигация (таб-бар, bottom sheet, нижний CTA), сквозной паттерн состояний
(Loading / Content / Empty / Error), tone of voice и общая микрокопия, глоссарий.

> **Объём визуальных требований.** Документы задают **функционально-структурные** требования
> (иерархия, компоненты, поведение, ограничения), без прескриптивных цветов/шрифтов — бренд
> в домене не зафиксирован; визуальный язык определяет дизайнер.

## Пользовательские сценарии (user flows)

- **Вход:** запуск → SCR-001 (телефон → код из SMS) → SCR-002.
- **Запись (UC-3):** SCR-002 → SCR-003 → SCR-004 → BS-002 → (SCR-005). После первой записи в BS-002 — запрос разрешения на push (Should, FR-20).
- **Фильтрация (UC-2):** SCR-002 → BS-001 → SCR-002 (с применёнными фильтрами).
- **Отмена (UC-4):** SCR-005 → SCR-006 → BS-003 → SCR-006 (с обновлённым статусом).
- **Оценка маршала (UC-6):** SCR-005 → SCR-006 (завершённая бронь) → BS-004 → SCR-006.
- **Отмена центром (UC-5):** push → SCR-005 / SCR-006 (статус «Отменён центром», причина).

## Тон, бренд, ограничения

- **Тон:** простой, прямой, дружелюбный, без давления и штрафов; обращение на «вы» —
  [00-foundations.md §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).
- **Бренд:** не зафиксирован в домене — визуальный язык за дизайнером; обязательны структурные
  ограничения (контраст, размеры, иерархия).
- **Mobile-first:** крупные тач-зоны, высокий контраст, читаемость на солнце у трассы (NFR-1).
- **Что НЕ делаем на старте:** онлайн-оплата, штрафы за поздние отмены, программа лояльности,
  авто-погода, интерфейсы маршала/владельца, SMS/email-уведомления — см.
  [domain-description.md](../1-elicitation/domain-description.md) → «Границы скоупа».
