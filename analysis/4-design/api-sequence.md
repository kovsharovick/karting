# Sequence-диаграмма сценария создания брони (createBooking)

> Описание логики взаимодействия между Мобильным приложением и API Бэкенда при оформлении группового или одиночного бронирования (Use Case: UC-3).

## Сквозные технические правила

1. Авторизация: Каждый запрос должен содержать валидный токен в заголовке Authorization: Bearer <token>. Возврат кода 401 приводит к сбросу сессии в приложении.
2. Защита от дубликатов (Идемпотентность): Для предотвращения повторного списания мест из-за сбоев сети мобильное приложение обязано генерировать UUIDv4 и передавать его в заголовке Idempotency-Key.
3. Атомарность транзакции: Проверка остатка картов и свободной экипировки выполняется сервером внутри единой изолированной транзакции базы данных во избежание овербукинга.
4. Расчет стоимости: Конечная стоимость рассчитывается на стороне сервера. Оплата происходит исключительно в офлайн-формате при визите в картинг-центр (наличные или перевод).

## Диаграмма последовательности

```mermaid
sequenceDiagram
    actor User as Клиент
    participant App as Приложение
    participant API as API Бэкенд (/bookings)

    Note over App: Пользователь выбрал количество мест<br/>и указал потребность в экипировке.<br/>Экран оформления записи.
    User->>App: Тап по кнопке "Подтвердить запись"
    App->>App: Блокировка интерфейса (Loader)<br/>Генерация Idempotency-Key (UUIDv4)

    App->>API: POST /api/v1/bookings<br/>Headers: Authorization, Idempotency-Key<br/>Body: { "slot_id": "uuid", "seats_count": int, "rental_gear_count": int }
    
    Note over API: Сервер открывает транзакцию:<br/>1. Проверка авторизации по токену<br/>2. Проверка статуса слота (не отменен ли треком)<br/>3. Проверка: seats_count <= slot.free_karts<br/>4. Проверка: rental_gear_count <= slot.free_rental_gear

    alt [Успешный сценарий] Код 201 Created
        API->>API: Фиксация брони в БД (status: active)<br/>Списание инвентаря слота<br/>Расчет price_total
        API-->>App: 201 Created<br/>Body: { "id": "uuid", "status": "active", "price_total": 4500.00, "created_at": "timestamp" }
        App-->>User: Переключение на экран "Запись оформлена"<br/>Показ итоговой стоимости и условий оплаты офлайн
    
    else [Ошибка: Конфликт мест] Код 409 Conflict
        Note over API: Места успели занять параллельно<br/>или запрос превышает остаток free_karts
        API-->>App: 409 Conflict<br/>Body: { "error_code": "karts_unavailable", "message": "Недостаточно картов", "free_karts": 2 }
        App->>App: Разблокировка интерфейса
        App-->>User: Варнинг: "Недостаточно свободных мест. Доступно для записи: 2"
    
    else [Ошибка: Конфликт экипировки] Код 409 Conflict
        Note over API: Свободных картов хватает,<br/>но клубных шлемов осталось меньше,<br/>чем запрошено в rental_gear_count
        API-->>App: 409 Conflict<br/>Body: { "error_code": "gear_unavailable", "message": "Недостаточно экипировки", "free_rental_gear": 1 }
        App->>App: Разблокировка интерфейса
        App-->>User: Варнинг: "Не хватает клубных шлемов. Измените параметры гостей на 'своя экипировка'"
    
    else [Ошибка: Слот недоступен] Код 410 Gone
        Note over API: Заезд был отменен администратором<br/>в существующей инфраструктуре из-за погоды
        API-->>App: 410 Gone<br/>Body: { "error_code": "slot_not_available", "reason": "cancelled_by_track" }
        App->>App: Удаление слота из локального кэша расписания
        App-->>User: Модальное окно: "Этот заезд отменен администрацией. Выберите другой слот."
    
    else [Ошибка: Слот уже стартовал] Код 422 Unprocessable
        Note over API: Время старта заезда уже наступило
        API-->>App: 422 Unprocessable<br/>Body: { "error_code": "slot_started", "message": "Заезд уже начался" }
        App->>App: Разблокировка интерфейса
        App-->>User: Варнинг: "Заезд уже начался — запись недоступна"
    
    else [Ошибка авторизации] Код 401 Unauthorized
        API-->>App: 401 Unauthorized
        App->>App: Сброс просроченного токена сессии
        App-->>User: Перенаправление на экран авторизации (Ввод телефона и SMS OTP)
    
    else [Сетевой сбой / Ошибка 5xx]
        API-->>App: Сетевой таймаут или 503 Service Unavailable
        App->>App: Разблокировка интерфейса
        App-->>User: Показ снэкбара "Ошибка сети. Попробуйте еще раз"<br/>(Повторный тап отправит тот же Idempotency-Key)
    end
```

## Спецификация ответов API для обработки на клиенте

| HTTP Статус | Код бизнес-ошибки | Описание сценария | Ожидаемая реакция UI-слоя приложения |
| :-- | :-- | :-- | :-- |
| 201 Created | — | Запись успешно создана на бэкенде. Ресурсы заблокированы под пользователя. | Перевод пользователя на экран успеха (Success State). Вывод итоговой стоимости price_total. |
| 409 Conflict | karts_unavailable | Количество запрашиваемых картов превысило доступный остаток на момент фиксации транзакции. | Обновить счетчик мест на экране, подсветить поле ввода красным цветом, показать ограничение. |
| 409 Conflict | gear_unavailable | Недостаточно единиц клубной экипировки (шлемов) на складе для данного времени. | Выдать подсказку: "Предложите части участников взять свой шлем, чтобы пройти регистрацию". |
| 410 Gone | slot_not_available | Клиент пытался записаться на слот, который администратор только что перевел в статус cancelled. | Показать системное уведомление, принудительно вернуть пользователя назад на экран расписания (календарь). |
| 422 Unprocessable | slot_started | Время старта заезда уже наступило. Запись невозможна. | Показать уведомление: "Заезд уже начался". Кнопка записи блокируется на экране деталей. |
| 401 Unauthorized | — | Токен сессии истек или недействителен. | Очистить локальное хранилище токенов, перенаправить на экран входа (SCR-001). |
| 5xx / Timeout | — | Внутренняя ошибка сервера или сбой сети. | Показать снэкбар с ошибкой. Повторный запрос использует тот же Idempotency-Key для защиты от дублей. |
