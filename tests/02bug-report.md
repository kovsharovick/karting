# 🐞 Баг-репорт: ошибка при фильтрации слотов по дате

**Дата:** 2026-07-05  
**Автор:** Разработчик  
**Статус:** ❌ Не исправлено  
**Приоритет:** Высокий (блокирует использование фильтра)

---

## 1. Краткое описание

При попытке отфильтровать слоты по дате (выбор даты в форме фильтров) возникает ошибка 500 с `ConversionFailedException`. Бэкенд не может преобразовать строку `2026-07-31` в `OffsetDateTime`.

---

## 2. Симптомы и шаги воспроизведения

**Симптомы:**
- При нажатии «Применить» в фильтрах с выбранной датой происходит внутренняя ошибка сервера.
- В логах видно:

Failed to convert from type [java.lang.String] to type [@org.springframework.web.bind.annotation.RequestParam java.time.OffsetDateTime] for value [2026-07-31]
java.time.format.DateTimeParseException: Text '2026-07-31' could not be parsed at index 2

- Страница показывает Whitelabel Error Page 500.

**Шаги для воспроизведения:**
1. Войти в приложение.
2. На странице `/slots` нажать «Фильтры».
3. Выбрать любую дату в поле «С» или «По» (например, 2026-07-31).
4. Нажать «Применить».
5. Происходит ошибка 500.

---

## 3. Ожидаемое поведение

- Фильтрация должна работать с выбранной датой, преобразуя её в `OffsetDateTime` с учётом часового пояса и времени (например, начало дня).
- Дата должна корректно передаваться в контроллер и обрабатываться `SlotService`.

---

## 4. Фактическое поведение

- В HTML-форме фильтров дата передаётся в формате `yyyy-MM-dd` (строка без времени).
- В контроллере `SlotController.listSlots` параметр `dateFrom` аннотирован `@DateTimeFormat(iso = DateTimeFormat.ISO.DATE_TIME)`, что требует полного ISO-формата (с временем и зоной).
- Spring не может распарсить `2026-07-31` в `OffsetDateTime`, потому что не хватает времени и смещения.

---

## 5. Причина (Root Cause)

**Несоответствие формата данных:**

- JSP-форма отправляет дату в виде строки `2026-07-31` (только дата).
- Бэкенд ожидает `OffsetDateTime` в формате `yyyy-MM-ddTHH:mm:ssZ` или `yyyy-MM-ddTHH:mm:ss+03:00`.
- Аннотация `@DateTimeFormat(iso = DateTimeFormat.ISO.DATE_TIME)` не умеет парсить только дату без времени.
