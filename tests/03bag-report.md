# 🐞 Баг-репорт: NullPointerException при открытии деталей брони

**Дата:** 2026-07-05  
**Автор:** Разработчик  
**Статус:** ❌ Не исправлено  
**Приоритет:** Высокий (блокирует просмотр деталей брони)

---

## 1. Краткое описание

При попытке открыть страницу деталей брони (переход по ссылке из списка `/bookings`) возникает ошибка 500 с `NullPointerException`. Причина – вызов метода `isAfter()` на объекте `OffsetDateTime`, который равен `null`. Ошибка возникает в JSP при попытке сравнить время старта заезда с текущим временем для отображения кнопки «Отменить».

---

## 2. Симптомы и шаги воспроизведения

**Симптомы:**
- После успешного создания брони переход на `/bookings` показывает список броней.
- При клике на любую бронь открывается страница с ошибкой 500 (Whitelabel Error Page).
- В логах видно:
  ```
  java.lang.NullPointerException: Cannot invoke "java.time.OffsetDateTime.toEpochSecond()" because "other" is null
  ```
  Указание на строку JSP, где используется `${booking.slot.startAt.isAfter(now)}`.

**Шаги для воспроизведения:**
1. Авторизоваться.
2. Создать бронь (записаться на слот).
3. Перейти в «Мои брони» (`/bookings`).
4. Нажать на любую бронь.
5. Отображается ошибка 500.

---

## 3. Ожидаемое поведение

- Страница деталей брони должна открываться без ошибок.
- Если слот или время старта отсутствуют (что в штатной работе невозможно), UI должен корректно обрабатывать такие случаи (например, показывать «Информация недоступна» или не отображать кнопки).
- Кнопка «Отменить» должна отображаться только для активных броней, у которых время старта ещё не наступило.

---

## 4. Фактическое поведение

- В JSP есть код:
  ```jsp
  <c:when test="${booking.slot.startAt.isAfter(now)}">
  ```
  При этом `booking.slot` или `booking.slot.startAt` может быть `null` (например, если слот был удалён или не подгрузился).
- Вызов метода `isAfter` на `null` приводит к `NullPointerException`.
- Ошибка возникает на этапе рендеринга JSP, поэтому страница не отображается.

---

## 5. Причина (Root Cause)

**Недостаточная проверка на null в JSP:**

- В `BookingService.getBooking` мы используем метод `bookingRepository.findByIdWithSlot(bookingId)`, который должен подгружать слот с помощью `JOIN FETCH`. Однако если по какой-то причине слот не найден (например, он был удалён из БД, но бронь осталась), поле `slot` в объекте `Booking` будет `null`.
- В JSP мы обращаемся к `booking.slot.startAt` без проверки на `null`, что и вызывает NPE.
- Также возможно, что `startAt` может быть `null` даже если `slot` не `null` (редкий случай, но теоретически возможен).
